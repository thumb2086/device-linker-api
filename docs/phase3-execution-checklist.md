# Phase 3 細化任務清單（建議執行版）

> 目標：將 main 分支核心娛樂功能（子熙賭場 + 排行榜 + VIP + 背包）完整遷移至新架構，並完成端到端整合驗證。

## 0) 執行策略與排序

### 建議優先順序
1. **Leaderboard（優先）**：影響主畫面，且可先把統計資料流打通。
2. **VIP + 背包（中段）**：模組相對獨立，可並行完成前後端骨架。
3. **Casino 遊戲（最後，分批）**：份量最大，依複雜度由簡到難遷移。

### 建議工期（7–10 天）
- Day 1–2：A 排行榜
- Day 3–4：B VIP + 背包
- Day 5–8：C Casino 遊戲遷移（分批）
- Day 9–10：D 整合測試、修補與上線前驗收

## 風險與建議

1. **排行榜查詢壓力風險**：先做索引與快取策略，避免功能上線後才補性能。
2. **遊戲遷移不一致風險**：先定義統一遊戲協議（下注/結算/事件），再遷移單一遊戲。
3. **VIP 與下注限制繞過風險**：必須在 API/Domain 層做硬性檢查，不可只靠前端。
4. **整合期回歸風險**：每天至少跑一次主流程 smoke（Lobby → Game → Wallet → Leaderboard）。
